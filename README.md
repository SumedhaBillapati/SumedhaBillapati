## 🧠 How I Think

> *"The data never lies, but it does leave things out. My job is to find what's missing."*

I'm a **5 Whys** person at heart. I once used it on a retail sales dataset where margins were going negative, and everyone assumed it was a pricing issue. Kept asking why. Turned out it was discounting specifically on *tables* that was eating into profits. One framework, one insight, completely different solution.

That habit carries into everything: PMO reporting, operations analysis, HR data, sustainability metrics. Before I build a dashboard or write a query, I want to understand the real question behind the question.
